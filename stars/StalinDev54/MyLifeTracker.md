# MyLifeTracker -- 数字生命 项目文档
> 个人生活数据可视化与 AI 数字生命辅助工具，专注于「实时活动追踪 + 效率管理 + 健康数据联动」，打造「有温度的全场景个人数据中枢」

## 一、项目概述
### 1.1 核心定位
- 面向个人用户的轻量级数据追踪工具，**以 AI 数字生命为核心交互载体**，实现活动数据的可视化、智能交互与个性化建议
- 支持多场景数据联动（应用、音乐、运动、健康），解决「数据分散、交互冰冷」痛点，满足效率管理与生活记录双重需求
- 核心差异：数字生命不仅是工具，更是「懂你的生活助手」，可基于数据主动提供提醒、分析与陪伴式交互

### 1.2 项目状态
- **开发阶段**：Demo 版本已完成核心功能验证，即将开源（当前进度：数字生命图像页面部署开发中，核心模块联调完成）
- **支持平台**：暂支持 Windows/macOS（桌面端 Demo）；后续计划适配 Linux 及 iOS/Android 移动端
- **协作邀请**：欢迎开发者点亮 ⭐ Star 支持，关注 [作者博客](https://blog.jiclub.site) 获取实时开发日志，第一时间参与开源协作

### 1.3 核心功能矩阵
| 功能分类       | 具体能力                                                                 | 数字生命联动亮点                          |
|----------------|--------------------------------------------------------------------------|-------------------------------------------|
| 实时活动追踪   | 应用使用时长统计、当前进程识别、网易云音乐播放状态（含「一起听」）同步    | 数字生命实时播报当前活动（如“已专注工作1小时”） |
| 效率管理       | 待办任务创建、专注状态切换、任务进度实时更新                              | 专注超时提醒、任务完成庆祝互动             |
| 运动数据联动   | 骑行轨迹记录（对接行者 API）、路径可视化、关键数据统计                    | 运动后生成健康建议（如“今日骑行30km，需补充水分”） |
| AI 数字生命交互| 音频对话、形象实时互动、语音通话、文本交互                                | 基于生活数据生成个性化陪伴内容（如周总结） |
| 数据稳定性     | 锁屏日志自动更新、断联数据补录                                          | 数据异常时主动提醒用户（如“锁屏记录未同步”） |

## 三、已实现/修复功能（已完成项）
- [x] **骑行轨迹功能**  
  - 技术细节：对接行者 API v2（OAuth2.0 授权），支持 GPX 轨迹解析、SVG 路径渲染与 ECharts 热力图展示  
  - 完成时间：2025年9月29日  
  - 功能亮点：轨迹回放、距离/速度/耗时统计、导出轨迹文件（GPX/PNG）
  - 数字生命联动：骑行结束后自动生成数据总结语音播报

- [x] **锁屏记录修复**  
  - 解决方案：监听系统锁屏事件（Windows：`powerMonitor` / macOS：`NSWorkspace`），触发日志异步写入  
  - 解决问题：修复锁屏状态下应用使用时长统计中断、数据缺失问题  
  - 优化点：断联后重新解锁自动补录缺失日志

- [x] **网易云音乐联动**  
  - 实现方式：集成 [NeteaseCloudMusicApi](https://github.com/Binaryify/NeteaseCloudMusicApi)，通过 WebSocket 实时推送播放状态/歌词  
  - 特殊适配：支持「一起听」模式用户状态同步（需登录态验证）、歌词逐句同步展示  
  - 数字生命联动：歌曲切换时主动播报歌曲名称、歌手，支持语音点歌

- [x] **待办专注功能**  
  - 核心逻辑：任务与专注状态绑定，专注时长计入任务进度，支持 Pomodoro 番茄钟模式（25min 专注+5min 休息）  
  - 交互设计：专注状态切换触发桌面通知+数字生命语音提醒，中断时弹窗确认  
  - 数据统计：生成每日/每周专注报告（专注时长、任务完成率）

- [x] **AI 数字生命集成**  
  - 技术依赖：MCP 服务处理音频流，对接 [小智 AI](https://xiaozhi.me) 实现语义理解，TTS/ASR 完成语音交互  
  - 功能支持：文本转语音（普通话/英语）、语音识别、实时语音通话、数字生命形象表情同步  
  - 交互亮点：支持自定义数字生命昵称、语速、语气，形象随语音实时动效

## 七、快速启动指南（Demo 版）
### 7.1 环境要求
- Node.js ≥ 18.17.0（建议用 nvm 管理版本）
- npm ≥ 9.6.0 或 yarn ≥ 3.6.0
- 系统：Windows 10+/macOS 12+

### 7.2 本地启动步骤
1. （开源后）克隆仓库：
   ```bash
   git clone https://github.com/StalinDev54/MyLifeTracker
   cd MyLifeTracker
   ```
2. 安装依赖：
   ```bash
   npm install
   # 或 yarn install
   ```
3. 配置第三方 API（可选，不配置不影响基础功能）：
   - 复制 `config/example.env` 为 `config/.env`
   - 填写行者 API Key、网易云音乐 Cookie（获取方式见 `docs/api-guide.md`）
4. 启动 Demo：
   ```bash
   npm run dev
   # 启动后自动打开前端界面，数字生命形象默认加载示例形象
   ```

## 九、常见问题（FAQ）
1. **Q：Demo 版数据存储在哪里？是否安全？**  
   A：Demo 版使用本地 JSON 文件存储（路径：`src/shared/data/`），所有数据仅保存在本地，无云端上传，安全性可控。

2. **Q：数字生命形象可以自定义吗？**  
   A：当前 Demo 版未提供默认形象，「自定义形象」功能正在开发中，开源后将优先支持贡献者体验。

3. **Q：没有相关开发经验，可以参与吗？**  
   A：完全可以！项目新手友好，提供入门任务和技术指导，文档优化、测试等角色无需深厚开发经验，欢迎学生/入门开发者加入。

4. **Q：项目开源后会商业化吗？**  
   A：项目采用 MIT 许可证，开源后允许非商业/商业使用（需保留版权声明），核心功能永久免费，未来可能推出付费增值服务（如高级形象、云端备份），但不影响基础功能使用。
